# ADR-0019 — ChatGPT Codex como agente: fluxo auditável de mudanças

Status: Accepted  
Data: 2026-02-10

## Contexto
O projeto usa agentes (ex.: ChatGPT Codex) para sugerir e implementar mudanças.
Sem um processo padronizado, fica difícil auditar:
- o que foi proposto
- o que foi alterado
- por que foi escolhido
- como foi verificado
- quais evidências suportam a mudança

## Decisão
Adotar um fluxo obrigatório e auditável para trabalho com agentes:

1) Toda tarefa do agente deve:
- declarar objetivo, escopo e restrições
- referenciar ADRs e docs impactados
- listar plano de execução

2) Toda execução deve produzir artefatos versionados em \docs/chatgpt_codex/runs/\:
- proposta (proposal)
- registro do que mudou (changes)
- verificação (verification: comandos/saídas)
- evidência (evidence: links/paths/logs)
- patch notes (resumo para auditoria)

3) O agente deve respeitar princípios do projeto:
- core vs stages
- determinismo
- contract-first
- evidence-first
- deny-by-default
- IA como consumidora (não decisora) quando aplicável

## Consequências
- Mudanças ficam rastreáveis e revisáveis
- Auditoria simplificada (artefatos por execução)
- Menor risco de regressões e decisões “opacas”
